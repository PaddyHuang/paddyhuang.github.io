3.2 队列

3.2.1 队列的基本概念

1. 定义

队列是一种操作受限线性表，只允许在表的一端进行插入，在表的另一端进行删除。
操作特性：先进先出

2. 基本操作

  1. InitQueue(&Q)InitQueue(\&Q)InitQueue(&Q)：初始化队列，构造一个空队。
  2. QueueEmpty(Q)QueueEmpty(Q)QueueEmpty(Q)：判空，空为true，非空为false。
  3. EnQueue(&Q,x)EnQueue(\&Q, x)EnQueue(&Q,x)：入队，队列未满，则将x加入。
  4. DeQueue(&Q,&x)DeQueue(\&Q, \&x)DeQueue(&Q,&x)：出队，队列非空，则删除队首元素。
  5. GetHead(Q,&x)GetHead(Q, \&x)GetHead(Q,&x)：读队首元素，队列非空，则将队首元素赋给x。

3.2.2 队列的顺序存储结构

1. 顺序队列

定义

cpp

顺序队列需要分配一块连续的存储单元存放数据元素，并附设队首指针front和队尾指针rear（也可以让front指向队首元素的前一个位置，rear指针指向队尾元素的后一个位置）。

  1. 初始状态：（队空条件）Q.front==Q.rear==0Q.front == Q.rear == 0Q.front==Q.rear==0；
  2. 进队操作：队列未满时，先送值到队尾元素，再将队尾指针加1；
  3. 出队操作：队列不空时，先取队首元素的值，再将队首元素加1.

缺点：存在假溢出现象。

2. 循环队列

定义

把存储队列元素的表从逻辑上看成一个环，即循环队列。当队首指针Q.front=MaxSize−1Q.front = MaxSize - 1Q.front=MaxSize−1后，再前进一个位置就自动到0，这可以利用除法取余运算来实现。

  1. 初始状态：Q.front==Q.rear==0Q.front == Q.rear == 0Q.front==Q.rear==0
  2. 队首指针进1：Q.front=(Q.front+1)Q.front = (Q.front + 1) % MaxSizeQ.front=(Q.front+1)
  3. 队尾指针进1：Q.rear=(Q.rear+1)Q.rear = (Q.rear + 1) % MaxSizeQ.rear=(Q.rear+1)
  4. 队列长度：(Q.rear+MaxSize−Q.front)(Q.rear + MaxSize - Q.front) % MaxSize(Q.rear+MaxSize−Q.front)
  5. 出入队时，指针都按顺时针方向进1

[Note] 如何判空及判满

  1. 牺牲一个单元来区分队空队满。入队时少用一个队列单元，这是一种较普遍的方法。约定以“队首指针在队尾指针的下一位置作为队满的标志”、
    • 队满条件： (Q.rear+1)(Q.rear + 1) % MaxSize == Q.front(Q.rear+1)
    • 队空条件：Q.rear==Q.frontQ.rear == Q.frontQ.rear==Q.front
    • 队列中的元素个数：(Q.rear−Q.front+MaxSize)(Q.rear - Q.front + MaxSize) % MaxSize(Q.rear−Q.front+MaxSize)
  2. 增设一个元素个数的数据成员。这样，则队空的条件为：Q.size==0Q.size == 0Q.size==0；队满的条件为：Q.size==MaxSizeQ.size == MaxSizeQ.size==MaxSize.
  3. 增设tag数据成员。当tag==0tag == 0tag==0时，若因删除导致Q.front==Q.rearQ.front == Q.rearQ.front==Q.rear，则队空；当tag==1tag == 1tag==1时，若因插入导致Q.front==Q.rearQ.front == Q.rearQ.front==Q.rear，则队满。

3.2.4 双端队列

定义

两端都可以进行入队出队操作的队列。

错题

一、单选

8.[2011] 已知循环队列存储在一维数组A[0…n-1]中，且队列非空时front和rear分别指向对头元素和队尾元素。若初始时队列为空，且要求第一个进入队列的元素存储在A[0]处，则初始时front和rear的值分别为（ ）。

A. 0,0
B. 0,n-1
C. n-1,0
D. n-1,n-1
[Analysis]
注意审题，首先，初始队列为空。而要求第一个进入队列的元素存储在A[0]处，想到入队只修改rear即可，所以rear=n-1，这样第一个元素入队，rear进1，就会到A[0]处了。front没有修改，一直为0.
∴B\therefore B∴B✔️

11. 最适合用作链队的链表是（ ）。

A. 带队首指针和队尾指针的循环单链表
B. 带队首指针和队尾指针的非循环单链表
C. 只带队首指针的非循环单链表
D. 只带队首指针的循环单链表
[Analysis]
队列需要双端操作，所以C、D不合适。A在完成出入队列之后，还要将其修改为循环，对于队列来说多余了。
对于B，头指针适合删除首结点；尾指针适合插入新结点。
∴B\therefore B∴B✔️

12. 最不适合用作链式队列的链表是（ ）。

A. 只带队首指针的非循环双链表
B. 只带队首指针的循环双链表
C. 只带队尾指针的循环双链表
D. 只带队尾指针的循环单链表
[Analysis]
由于非循环双链表只带队首指针，在入队时，需要修改尾指针的指针域，而查找尾指针需要耗时O(n)O(n)O(n)。B、C、D都可以在O(1)O(1)O(1)时间内找到队首和队尾。
∴A\therefore A∴A✔️

13. 在用单链表实现队列时，队头在链表的（ ）位置。

A. 链头
B. 链尾
C. 链中
D. 以上都可以
[Analysis]
由于在队头做出队操作，为了方便删除对头元素，故总是选择链头作为队头。
∴A\therefore A∴A✔️

14. 用链式存储方式的队列进行删除操作时需要（ ）。

A. 仅修改头指针
B. 仅修改尾指针
C. 头尾指针都要修改
D. 头尾指针可能都要修改
[Analysis]
链队删除元素时，一般情况下，只需修改头指针即可。但是如果此时队列中仅有一个元素，则尾指针也需要被修改，因为仅有一个元素时，删除后队列为空，所以需要修改尾指针为rear = front。
∴D\therefore D∴D✔️

16. 假设循环单链表表示的队列长度为n，队头固定在链表表尾，若只设头指针，则进队操作的时间复杂度为（ ）。

A. O(n)O(n)O(n)
B. O(1)O(1)O(1)
C. O(n2)O(n^2)O(n2)
D. O(nlog⁡2n)O(n\log_2n)O(nlog2​n)
[Analysis]
由题知，队头在表尾，只有头指针。所以，出队在队头（表尾），入队在队尾（表头）。而只有头指针，每次进队修改头指针的同时，为了保持循环，还有修改尾结点，故需要耗时O(n)O(n)O(n)去寻找尾结点。
∴A\therefore A∴A✔️
