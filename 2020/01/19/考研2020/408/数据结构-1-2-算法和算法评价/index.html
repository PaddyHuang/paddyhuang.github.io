1.2 算法和算法评价

1.2.1 算法的基本概念

算法是对特定问题求解步骤的一种描述，具有以下5个重要特性：
1. 有穷性
2. 确定性
3. 可行性
4. 输入
5. 输出

设计一个好的算法，考虑以下目标：
1. 正确性
2. 可读性
3. 健壮性
4. 效率与低存储量需求

1.2.2 算法效率的度量

时间复杂度——算法中所有语句频度之和T(n)T(n)T(n)与基本运算的频度O(f(n))O(f(n))O(f(n))同数量级。

  1. 一个语句的频度——该语句在算法中被重复执行的次数
  2. 最坏时间复杂度——最坏情况下，算法的时间复杂度。（默认考虑）
  3. 平均时间复杂度——所有可能输入实例在等概率出现的情况下，算法的期望运行时间。
  4. 最好时间复杂度——最好情况下，算法的时间复杂度。
  5. 常见的渐近时间复杂度有：

⋆O(1)<O(log⁡2n)<O(n)<O(nlog⁡2n)<O(n2)<O(n3)<O(2n)<O(n!)<O(nn)\star \quad O(1) < O(\log_2{n}) < O(n) < O(n\log_2{n}) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n) ⋆O(1)<O(log2​n)<O(n)<O(nlog2​n)<O(n2)<O(n3)<O(2n)<O(n!)<O(nn)

空间复杂度——该算法所耗费的存储空间，是问题规模n的函数

  1. 算法原地工作——算法所需辅助存储空间是常量，即O(1)O(1)O(1)。

一、单选

1. 一个算法应该是（ ）。

A.程序B.问题求解步骤的描述C.要满足5个基本特征D.A和CA.\text{程序}\quad B.\text{问题求解步骤的描述}\quad C.\text{要满足5个基本特征}\quad D.\text{A和C} A.程序B.问题求解步骤的描述C.要满足5个基本特征D.A和C

[解]
程序不一定有穷，如死循环、操作系统等，而算法必须有穷。
算法代表了对问题求解步骤的描述，而程序则是算法在计算机上的体现。

∴B is true.\therefore\text{B is true.} ∴B is true.
